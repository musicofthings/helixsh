helixsh

Bioinformatics-first AI shell wrapper for Nextflow and nf-core — zero dependencies, dry-run safe, RBAC enforced.

Get Started View on GitHub Command Reference
$ pip install -e . && helixsh doctor

Features

50+ commands covering every stage of a bioinformatics workflow — from intent parsing to cloud cost estimation and audit trails.

🧬

Intent Parsing

Natural-language → Nextflow plan. Say "run nf-core rnaseq on tumor-normal samples" and get a ready command.

🚀

Pipeline Execution

Run nf-core pipelines with Docker, Podman, Singularity, Apptainer, or Conda. Dry-run by default — add --execute to go live.

📋

Samplesheet Tools

Validate existing CSVs or auto-generate them by scanning a FASTQ directory for R1/R2 pairs.

🗂️

Reference Genomes

Bioconda Integration

Installation

Three ways to get helixsh — choose what fits your workflow. No PyPI dependencies required.

Option 1 — pip editable install

Best for development and daily use.

git clone https://github.com/musicofthings/helixsh
cd helixsh
python -m venv .venv
source .venv/bin/activate
pip install -e .
helixsh doctor

Option 2 — single-file .pyz

Copy the zipapp anywhere on $PATH — no venv needed.

./scripts/package_local.sh
./dist/helixsh.pyz doctor

# Copy to PATH:
cp dist/helixsh.pyz ~/.local/bin/helixsh

Option 3 — run without installing

Use Python module invocation directly from the repo.

git clone https://github.com/musicofthings/helixsh
cd helixsh
python -m helixsh doctor

Quick Start

From zero to a running pipeline in five steps.

1
Check your environment
helixsh doctor
2
Parse natural-language intent into a plan
helixsh intent "run nf-core rnaseq on tumor-normal samples using docker"
3
Generate a samplesheet from your FASTQ files
helixsh samplesheet-generate --fastq-dir /data/fastq --pipeline rnaseq --out samplesheet.csv
4
Estimate cloud cost before running
helixsh cost-estimate --cpu 32 --memory-gb 128 --hours 6 --provider aws
5
Run the pipeline (dry-run first, then add --execute)
# Preview the command
helixsh run nf-core/rnaseq --runtime docker --input samplesheet.csv --resume

# Run for real
helixsh run nf-core/rnaseq --runtime docker --input samplesheet.csv --resume --execute

Command Reference

All commands support --help. Global flags: --strict (require explicit confirmation) and --role auditor|analyst|admin (RBAC).

🚀 Core Execution

runRun an nf-core pipeline via Nextflow with container runtime selection. Dry-run by default; add --execute to run for real.
doctorCheck all required tools (nextflow, java, docker, conda…) and print their versions.
planDisplay planning guidance for building a Nextflow command.
posix-wrapRender an explicit POSIX exec sh -c boundary wrapper around a command.
preflightRun all pre-flight checks in one shot: schema, params, workflow, samplesheet, config, image digest.

🧬 Intent Parsing

intentMap a natural-language request to a concrete Nextflow plan. Detects assay, container runtime, and pipeline.
profile-suggestSuggest a pipeline and Nextflow profile arguments based on --assay and --reference.

📦 nf-core Pipelines

nf-launchLaunch a pipeline using nextflow launch or Seqera Platform with full param support.
nf-authShow Seqera Platform authentication status (TOWER_ACCESS_TOKEN).
pipeline-listList known pipelines with latest versions; --refresh to fetch from nf-core API.
pipeline-updateCheck if a pinned pipeline version matches the latest release.

📋 Samplesheet Tools

samplesheet-validateValidate an nf-core samplesheet CSV for required columns, file existence, and data integrity.
samplesheet-generateAuto-generate an nf-core samplesheet by scanning a FASTQ directory for R1/R2 pairs.

🗂️ Reference Genomes

ref-listList all reference genomes in the built-in catalogue (GRCh38, GRCh37, GRCm39, TAIR10…).
ref-downloadDownload and cache a reference genome FASTA + GTF from Ensembl with SHA-256 checksums.

🐍 Bioconda Integration

📊 Trace & Cost Analysis

☁️ Seqera Platform (Tower)

RBAC Role-Based Access Control

Pass --role <role> to enforce access controls. Default role is analyst.

RolePermitted OperationsRestricted Operations
auditor Read-only: doctor, nf-list, pipeline-list, ref-list, trace-summary, audit-log, provenance-show, diag All write/execute commands blocked
analyst Everything auditor can do, plus: run, intent, samplesheet-*, ref-download, tower-submit, tower-cancel, cost-*, mcp-* conda environment management, security scanning
admin All commands including: conda-install, conda-env-create, sbom-generate, secret-scan, audit-verify, provenance-export None 
# Run as auditor (read-only)
helixsh --role auditor audit-log

# Run as admin to install a package
helixsh --role admin conda-install samtools --execute

# Strict mode: require --execute and --yes everywhere
helixsh --strict --role analyst run nf-core/rnaseq --runtime docker --input s.csv

Environment Variables & Configuration

helixsh reads these environment variables at startup. All are optional unless the corresponding feature is used.

TOWER_ACCESS_TOKEN
Seqera Platform personal access token. Required for tower-* commands.
TOWER_API_ENDPOINT
Seqera Platform API base URL. Defaults to https://api.cloud.seqera.io. Override for self-hosted deployments.
TOWER_WORKSPACE_ID
Seqera Platform workspace ID. Used by tower-submit, tower-list, and tower-envs.
HELIXSH_AUDIT_FILE
Path to the HMAC-signed JSONL audit log. Defaults to ~/.helixsh/audit.jsonl.
HELIXSH_PROVENANCE_DB
Path to the SQLite provenance database. Defaults to ~/.helixsh/provenance.db.

Example shell setup

# ~/.bashrc or ~/.zshrc
export TOWER_ACCESS_TOKEN="your-token-here"
export TOWER_WORKSPACE_ID="12345678"
export HELIXSH_AUDIT_FILE="$HOME/.helixsh/audit.jsonl"
export HELIXSH_PROVENANCE_DB="$HOME/.helixsh/provenance.db"
