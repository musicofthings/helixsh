Design

Built on five principles

Every design decision in helixsh follows from these.

🔒

POSIX execution boundary

Final execution is always real sh/bash commands. No proprietary runtime, no hidden behavior.

⚡

Nextflow is the authority

helixsh plans, validates, explains, and diagnoses. It does not replace Nextflow or any workflow engine.

📦

Container-first execution

Docker, Podman, Singularity, and Apptainer are enforced. Host binaries are blocked. Image digests are preferred.

🤖

LLM as planner, not executor

AI proposes plans and fixes. helixsh validates and decides. Nothing executes without your approval.

🌐

Offline-capable by design

Local reasoning, cached schemas, and optional internet access. Works on air-gapped HPC clusters.

🛡️

Dry-run by default

Every command shows what would run before it runs. --execute is always explicit. Strict mode requires --yes too.

Capabilities

Everything a bioinformatician needs

From samplesheet generation to cloud cost estimation — all in one tool.

intent

Natural Language Planning

Describe your analysis in plain English. helixsh maps it to Nextflow parameters, detects assay type, runtime, and sample model.

  • WGS, WES, RNA-seq, cfDNA, ChIP-seq, scRNA-seq
  • Tumor/normal, trio, cohort sample models
  • Auto-selects pipeline profile and flags
validation

Pre-flight Validation

Run a full suite of checks before touching a cluster — schema, container policy, offline readiness, context, and image digest.

  • nf-core JSON schema validation
  • Nextflow process AST parsing
  • Samplesheet column and type checks
  • Container digest enforcement
diagnosis

Failure Diagnosis

helixsh reads exit codes and memory constraints to explain why a process failed and suggests concrete remediation steps.

  • Exit code 137 → OOM → increase memory or switch tool
  • Cache invalidation analysis
  • Resume efficiency report
resources

Resource & Cost Planning

Estimate CPU and memory before you submit. Fit empirical calibration multipliers from past runs. Compare cloud providers.

  • Tool-aware estimates (STAR, Salmon, GATK, BWA…)
  • AWS / GCP / Azure cost comparison
  • Snakemake calibration import
audit

Audit & Compliance

Every run writes a tamper-evident audit log with reproducible SHA-256 hashes. Suitable for clinical genomics and regulated HPC.

  • HMAC-signed audit trail
  • RBAC enforcement (analyst / auditor / viewer)
  • Reproducible execution hashes
  • SQLite provenance database
platform

Seqera Platform Integration

Launch, monitor, and manage pipelines on Seqera Platform (formerly Nextflow Tower) via both the CLI and REST API.

  • nf-launch and tower-submit
  • Run status polling
  • Compute environment listing
hpc

HPC / Environment Modules

Generate modules.config for Lmod/Environment Modules clusters. No Docker required on HPC.

  • Known module names for 20+ bioinformatics tools
  • Process selector prefix support
  • Singularity / Apptainer mode
ai

MCP Gateway + Claude Code

AI proposes diffs and config changes via a controlled gateway. helixsh approves and executes. Claude never touches your filesystem directly.

  • Proposal / approve / execute workflow
  • Capability allowlist enforcement
  • Multi-agent arbitration (HAPS v1)

Installation

Get up and running

Requires Python ≥ 3.10. No external dependencies beyond the standard library.

1

Clone the repo

Clone from GitHub and enter the directory.

git clone https://github.com/musicofthings/helixsh
cd helixsh
2

Create a virtualenv

Create and activate a Python virtual environment.

python -m venv .venv
source .venv/bin/activate
# Windows: .venv\Scripts\activate
3

Install

Install helixsh in editable mode.

pip install -e .
4

Verify

Run the environment check.

helixsh doctor

Or use the self-contained executable

./scripts/package_local.sh
./dist/helixsh.pyz doctor

The .pyz archive is a self-contained Python zipapp — no venv, no PyPI, no internet required.
